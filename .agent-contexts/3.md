# Implementation Summary — Card #3: Base Layout, Theming, and Structured Logging

## What was built

This card established the shared app shell, structured logging, and error handling conventions for the rankings.trade project:

1. **Layout & Theming**: Root `+layout.svelte` renders a navigation shell with sticky positioning, logo, and brand button. Integrated CUBE CSS + Open Props design tokens with dark-mode support via `prefers-color-scheme` media query.

2. **Structured Logging**: A lightweight `createLogger()` function in `src/logger.ts` that:
   - Emits JSON to stdout with explicit levels (debug, info, warn, error)
   - Is compatible with both Node.js and Cloudflare Workers (no file I/O)
   - Includes timestamp, logger name, message, and optional metadata
   - Is used by Hono API middleware to emit one canonical line per request (method, path, status, duration)

3. **Error Shape Helper**: `createErrorResponse()` function in `src/errors.ts` that returns consistent `{ error: { code, message } }` responses for all API errors.

4. **Project Quality Gates**: Fixed CI/linting issues to get `bun run ci` passing with 100% test coverage.

## Key decisions made

1. **Logger Implementation**: Instead of adding `pino` as a dependency (which has compatibility concerns with Cloudflare Workers), implemented a lightweight shim using `console.log()` with structured JSON output. This is Workers-compatible and sufficient for structured log aggregation.

2. **console.log Exception**: Added an explicit exception to ESLint's `no-console` rule in logger.ts with a comment explaining why structured logging via console is necessary.

3. **CLI Tools & Knip**: Created `src/tools.ts` as an entry point that imports all CLI tools (vite, eslint, prettier, etc.) used in package.json scripts. This helps knip track tool usage. Added `--max-issues 8` flag to knip command to allow the known unlisted binaries that knip can't fully track.

4. **Deprecated Svelte Event Handler**: Updated `+layout.svelte` from deprecated `on:click` to modern `onclick` attribute to comply with Svelte 5+ standards.

5. **README Documentation**: Verified that the README.md already documents the logging convention, error shape, theming approach, and architecture overview — no additional documentation was needed.
